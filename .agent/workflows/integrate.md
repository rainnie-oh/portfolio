# Integrate / Merge (代码整合模式)

**触发时机**：我说 "Integrate", "Merge", 或发给你一个完整外部代码包、或要求你把某段非标交互并进来。

**核心规矩：绝对不许直接改代码，必须先做 Merge Plan。**

请以清晰结构输出包含以下节点的 Merge Plan：
1. **New**：会新增哪些文件/组件（预告影响面）。
2. **Replace**：会覆盖原本项目里的哪些内容。
3. **Conflict**：可能发生的冲突点预测（样式/组件/逻辑交互/包依赖）。
4. **Minimal-risk path**：给出最小风险合并路径。（通常是：1️⃣ 先隔离接入尝试 → 2️⃣ 跑通后抽取复用逻辑 → 3️⃣ 最后再清退废弃代码）。
5. **Open decisions**：列出在这其中需要我优先去选的点。

**合并原则：**
只要有和 DS 体系(Design System)重合的部分：强制按 DS 改写/适配，绝不为了图快照搬外部代码片段！
